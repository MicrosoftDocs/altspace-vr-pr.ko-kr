# <a name="host-tools-for-frontrow-events"></a>FrontRow 이벤트용 호스트 도구

이 문서는 이벤트에 대해 FrontRow를 사용하도록 설정한 것을 기반으로 합니다. 이벤트에 FrontRow를 사용하도록 설정하려면 FrontRow 기능을 사용하여 대상 그룹 크기 조정 문서를 [참조하세요.](../faqs/scaling-audiences.md)

FrontRow가 아닌 이벤트에 대한 호스트 도구에 대해 알아보려면 [호스트 도구 개요를](host-tools-overview.md) 참조하세요.

이제 이벤트를 FrontRow로 설정했으므로 FrontRow에서 사용하도록 설정된 모든 멋진 기능을 사용하는 방법을 진행해보겠습니다.

처음에 이벤트를 만들 때 해당 이벤트에 대한 호스트 '역할'을 설정했습니다. 이벤트를 실행하는 데 도움이 하려는 다른 사람에게 호스트 역할을 제공할 수도 있습니다. 호스트 역할은 사용자와 동일한 호스팅 능력을 갖습니다(역할: [이벤트, 세계, 그룹 및 AltspaceVR의 역할](../getting-started/roles.md)참조). 호스트 역할은 라이브 가상 이벤트 호스팅을 간단하고 쉽게 만들기 위해 특별히 만든 특수 기능 및 기능에 대한 액세스 권한을 부여합니다.

다음은 AltspaceVR에서 FrontRow 이벤트를 호스트하기 위한 이러한 기능, 기능 및 기타 유용한 힌트에 대한 자세한 개요입니다.

## <a name="host-tools-overview"></a>호스트 도구 개요

호스트 도구는 보기 필드의 오른쪽 아래에 나타납니다. 이벤트를 만든 직후에 사용할 수 있으므로 언제든지 이벤트를 입력하고(이벤트 시작 시간은 이벤트의 호스트 또는 중재자에 적용되지 않음) 쇼타임 전에 도구에 익숙해질 수 있습니다. 호스트 도구는 대기 중(FrontRow 이벤트 실행에 필요), 대상 그룹 음소거 및 메시지 보내기, 스테이지 방해 요소 켜기/끄기, 메가포니핑, 게스트가 질문하거나 피드백을 제공하기 위해 "손을 들 수 있도록" 허용, FrontRow Room에서 Room로 이동하는 기능 등과 같은 항목을 빠르게 클릭할 수 있게 해줍니다.

![사용 가능한 작업이 강조 표시된 호스트 도구 메뉴가 열립니다.](images/host-front-row-img-01.png)

*GO ON-AIR 단추가 표시되면 이벤트가 FrontRow로 설정되었는지 알 수 있습니다.*

### <a name="toggle-mute-all"></a>Mute-All 전환

참석자를 음소거하면 방해가 되는 소리를 제한하고 FrontRow 이벤트에서 멋진 것을 유지하고 계속 사용할 수 있습니다. 음소거 함수는 호스트가 허용하는 한 또는 호스트가 공간을 벗어나는 동안 지속됩니다. 참석자가 이벤트에 들어가거나 음소거를 해제하려고 하면 참석자는 호스트에서 음소거되었다는 것을 알리는 화면 메시지를 받게 됩니다. 호스트가 음소거를 해제하면 참석자는 자신이 음소거되지 않았다는 메시지를 받게 됩니다.

### <a name="message-all"></a>메시지 모두

호스트는 FrontRow 이벤트에 있는 모든 참석자에게 메시지를 보낼 수 있습니다. 빠른 메모를 사용하거나 원래 메모를 작성합니다. 전송되면 모든 사람이 팝업으로 메시지를 받게 됩니다.

### <a name="amplify-my-voice"></a>내 음성 강화

호스트는 마이크를 강화하여 프런트 행 이벤트의 모든 사람이 거리에 관계없이 사용자의 의견을 들어줄 수 있도록 할 수 있습니다.

### <a name="go-on-air"></a>Go On-Air

호스트 역할이 있는 모든 사용자가 이 단추를 볼 수 있습니다. 켜기 이동을 클릭하면 모든 FrontRow 룸에 대한 아바타가 '미러링'됩니다. '미러'되면 프런트로 이벤트의 모든 인스턴스에서 모든 말과 작업을 볼 수 있습니다.

FrontRow 이벤트 전체에 대해 대기하는 것은 매우 중요합니다. 호스트 도구가 모든 FrontRow Room에서 제대로 작동하려면 대기 중이어야 합니다. 그렇지 않으면 호스트 도구가 있는 실내에서만 작동합니다.

> [!NOTE]
> 이벤트에 다시 참가해야 하는 경우,체방화소로 이동한 후 돌아가서 인터넷 연결을 끊고, 실내에서실로 이동합니다. 다시 돌아오면 대기 중이 아닙니다. 돌아가면 다시 Go On-Air를 클릭해야 합니다.

### <a name="toggle-stage-blocker"></a>단계 차단 토글

참석자가 대기할 수 있는 시기와 위치를 결정합니다. 스포트라이트를 표시하거나, 차례로 발표하거나, 그룹식 파티의 스테이지에 있는 모든 사람을 초대하도록 선택합니다.

> [!IMPORTANT]
> 모든 세계에 스테이지 방해가 있는 것은 아니므로 이벤트 템플릿을 정확하게 선택합니다. 공식 이벤트를 만들기 전에 테스트 이벤트에 대한 이벤트 템플릿을 숙지하세요.

## <a name="guest-panel"></a>게스트 패널

이벤트의 '방'에서 모든 사람을 추적할 수 있습니다. FrontRow 이벤트에는 여러 개의 방이 있으며, 게스트 패널에는 현재 있는 방의 게스트만 표시됩니다. 모든/마이크 열기/호스트/친구/대기 중을 클릭하여 표시된 항목을 필터링할 수 있습니다. 이 목록은방에 있는 사람에 대한 다중 페이지 분석을 제공하는 대화형 목록입니다.

잘못된 오디오를 가진 사람을 찾으려고 합니까? 마이크를 열어 누가 처리되지 않는지 확인합니다. 필요한 모든 사람이 FrontRow 이벤트에서 미러되도록 하고 싶으신가요? 현재 대기 중이고 미러된 모든 사람을 보려면 켜기를 선택합니다. 

### <a name="moderation-tools"></a>조정 도구

이벤트 호스트는 다른 사람의 환경을 관리해야 합니다. 효율적으로 조정하려면 사용자와 중재자에게 강력한 조정 도구가 필요합니다. FrontRow 이벤트의 호스트로 중단된 개인을 음소거하고, 이벤트(일시적으로 또는 이벤트 중)에서 규칙 분리기를 시작하고, 이벤트 중에 Moderator/호스트 역할을 다른 사용자에게 할당하여 대상 그룹 관리에 도움을 줄 수 있습니다.

게스트 패널에서 이름 옆에 있는 화자 아이콘을 클릭하거나 추가 옵션을 위해 게스트 패널의 오른쪽에 있는 햄버거 메뉴를 선택하여 게스트 패널에서 즉시 게스트를 음소거할 수 있습니다.

![게스트 패널에서 선택한 단일 게스트 항목](images/host-front-row-img-03.png)

*햄버거 메뉴를 클릭하여 모든 조정 도구 옵션을 엽니다.*

![조정 도구 옵션이 강조 표시된 단일 게스트 항목 선택](images/host-front-row-img-04.png)

이제 FrontRow 호스팅 매직을 사용하고 참석자에서 이러한 기능을 사용할 수 있습니다.
    * **메시지** - 일반적으로 다른 사용자에게 메시지를 보낼 수 있으려면 친구이어야 하지만 호스트는 이벤트의 모든 사용자와 메시지를 주고받을 수 있습니다.
    * **음소거** - 이벤트의 모든 사람을 위해 음소거합니다.
    * **메가phone** - 내 음성을 강화하는 기능을 제공합니다.
    * **대기** 중 - 이 게스트를 '미러'할 수 있습니다. 이렇게 하면 FrontRow 이벤트의 모든 방에서 모든 사람이 아바타를 보고 들었습니다.
    * **스테이지 액세스** - 스테이지 블로커를 통과하여 스테이지 영역으로 이동하도록 합니다.
    * **호스트 만들기** - FrontRow 이벤트에서 호스트 역할을 제공합니다. 즉석에서 Co-Host 추가하고, 온-에어로 이동하여 사물을 실행하거나 다른 방으로 이동하여 확인하도록 할 수 있습니다. 모든 사람이 대기 중이며 보조 회의실에 표시되어 있나요? 프레젠테이션이 모든 활성 회의실에서 동기화되어 있나요? 다른방에 있을 때 에어로 이동하게 하면 아바타와 Room에서 Room로의 아바타에 대해 말할 수 있습니다.
    * **Make Moderator** - 프런트 행 이벤트에서 Moderator 역할을 제공하여 컨트롤을 제어하는 데 도움을 줍니다. 중재자는 제한된 호스트 도구에 액세스할 수 있으므로 실내에서실로 이동하여 사물을 확인할 수 있지만, 온-에어로 이동할 필요는 없습니다. 
    * **Kick** - 일시적으로 또는 이벤트 중에 규칙 차단기를 이벤트에서 시작합니다.

## <a name="participation-panel"></a>참가 패널
 
이 패널을 사용하여 이벤트 참석자와 상호 작용할 수 있습니다. 질문&A가 있거나, Open Mic에 대해 수행할 큐를 구성하거나, 대화하는 게스트도 미러되도록 할 수 있습니다. 따라서 모든 사람이 모든 FrontRow 룸에서 서로 대화하는 것을 모두들을 수 있습니다.

![참가 탭이 선택된 호스트 패널 열기](images/host-front-row-img-05.png)

참가 패널은 대상의 상호 작용 및 참여를 관리하는 데 도움이 되도록 설계되었습니다. 여기서는 대상 그룹 참여의 잠금을 해제하기 위해 "손 발생 허용" 옵션을 전환할 수 있습니다. **이를 활성화하려면 FrontRow 이벤트에서 먼저 대기 중이어야 합니다. 그렇지 않으면 이 단추가 회색으로 표시됩니다.**

"손 발생 허용" 함수가 활성화되면 FrontRow 이벤트의 모든 사용자에 대한 "손 발생" 단추가 보기 필드의 오른쪽 아래 섹션에 표시됩니다.

![참가 패널에 강조 표시된 손 발생 함수](images/host-front-row-img-06.png)

*'손 발생'을 클릭하면 게스트가 제어하는 큐에 놓입니다.*

게스트가 이 단추를 클릭하면 참여 패널에 표시됩니다.

![큐에 손을 올려 게스트가 있는 참가 패널이 열립니다.](images/host-front-row-img-07.png)

추적할 수 있도록 참여 패널은 호출 대기 중인 게스트가 있을 때 빨간색 알림을 표시하고 각 게스트가 대기한 시간도 알려 줍니다.

이제 'On-Air + Megaphone' 단추를 클릭하여 호출할 수 있습니다. 활성화되면 단추가 파란색으로 바뀝니다. 이 작업은 두 개의 메시지를 손 발생기로 보내 마이크의 음소거를 해제하고 '호스트가 호출하여 더 크게 만들었습니다.'라는 메시지를 확인합니다. 수락하면 단추가 파란색으로 바뀌고, 모든 이가 보고 듣고 볼 수 있도록 '미러된' 켜기 및 내 음성 '메가phoned'를 강화합니다.

상호 작용을 마쳤으면 먼저 파란색 단추를 다시 클릭하여 끄기로 가져갔습니다. 단추는 더 이상 파란색이 아니고 두 번째 단추는 X를 클릭하여 목록에서 해제합니다. 이제 :) 등과 같은 2번에 대해 복습할 준비가 되었습니다.

임시 단계 액세스를 제공하거나 오른쪽의 모두 해제 단추를 클릭하여 손 발생기 목록을 완전히 지우고 새로 시작할 수도 있습니다.

## <a name="room-panel"></a>실내 패널

FrontRow 이벤트 중에 룸을 관리하는 데 필요한 정보/단추를 빠르게 살펴봅니다.

![호스트 패널에서 열린 방 패널 탭](images/host-front-row-img-08.png)

*현재 있는 방인 **(여기 있습니다.)라고** 표시됩니다.*

공간에는 공간 다시 설정, 공간 삭제, 사용자 재배포 및 원격 전송 단추가 있습니다.

![실내 패널에 강조 표시된 공간 다시 설정 단추](images/host-front-row-img-09.png)

현재 있는 공간에서 '공간 다시 설정'을 수행합니다. 한 번에 모든 방의 공간을 다시 설정할 수 없으며, 다시 설정을 수행하려면 각실로 이동해야 합니다.

![공간 패널에 강조 표시된 공간 삭제 단추](images/host-front-row-img-10.png)

현재 있는 공간을 영구적으로 삭제합니다. 삭제된 Room의 모든 사용자가 자동으로 이벤트에 다시 참가하고 다른 Room에 조인합니다.

![실내 패널에 강조 표시된 사용자 재배포 단추](images/host-front-row-img-11.png)

경우에 따라 FrontRow 이벤트가 잠시 동안 켜진 후 처음 몇 개의 방은 거의 용량에 있지만 마지막 방 또는 두 개의 객실에는 Room당 적은 수의 게스트만 있을 수 있습니다. 이 단추를 클릭하면 해당 Room의 게스트에게 현재 객실에 유지하거나 더 채워진 공간으로 가는 옵션이 제공됩니다.

> [!NOTE]
> 재배포하려는 공간으로 이동하지 않고도, 객실 패널에서 바로 모든 방의 사용자 재배포 단추를 클릭할 수 있습니다. 동일한 방으로 이 작업을 너무 많이 수행하지 않도록 합니다. 이 방의 게스트는 소수의 사람들과 함께 있을 수 있으며, 공간을 이동하라는 메시지를 계속 받지 않으려고 할 수 있습니다.

![실내 패널에 강조 표시된 원격 이동 단추](images/host-front-row-img-12.png)
